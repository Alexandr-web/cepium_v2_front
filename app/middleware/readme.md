# Middleware в Nuxt 4

Папка `app/middleware/` содержит навигационные стражники (middleware), которые выполняются перед переходом на определённую страницу или группой страниц. Они позволяют контролировать доступ, перенаправлять пользователей, проверять аутентификацию и т.д.

## Основные концепции

- **Глобальный middleware** – выполняется при каждом изменении маршрута.
- **Именованный middleware** – применяется к конкретным страницам через `definePageMeta()`.
- **Анонимный middleware** – определяется прямо в компоненте страницы (реже).

## Создание middleware

Файлы в этой папке автоматически загружаются Nuxt. Каждый файл должен экспортировать функцию (или объект с `default`).

### Пример: `auth.ts`

```ts
export default defineNuxtRouteMiddleware((to, from) => {
  const user = useUserStore(); // например, хранилище Pinia
  if (!user.isLoggedIn) {
    return navigateTo('/login');
  }
});
