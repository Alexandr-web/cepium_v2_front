# Layouts в Nuxt 4

Папка `app/layouts/` предназначена для хранения шаблонов (макетов) страниц. Layout определяет общую структуру приложения: хедер, футер, сайдбар, навигацию – то, что повторяется на нескольких страницах. Nuxt автоматически подгружает layout по умолчанию (`default.vue`) и позволяет переключать макеты на уровне страниц.

## Основные концепции

- **Единый макет по умолчанию** – `layouts/default.vue` используется для всех страниц, если не указано иное.
- **Именованные макеты** – можно создавать несколько макетов (например, `auth.vue`, `dashboard.vue`) и назначать их страницам через `definePageMeta`.
- **Вложенные макеты** – один макет может рендерить другой (например, внутри `default.vue` использовать `<NuxtLayout name="sub">`).
- **Автоимпорт** – layout-компоненты автоматически доступны по имени файла.
- **SSR-поддержка** – макеты рендерятся на сервере и гидратируются на клиенте.

## Структура папки

- layouts/
  - default.vue # основной макет (используется по умолчанию)
  - auth.vue # макет для страниц авторизации (без хедера/футера)
  - dashboard.vue # макет для панели управления (с сайдбаром)
  - empty.vue # пустой макет (без обёрток)

## Создание макета

### default.vue – базовый макет

```vue
<template>
  <div>
    <OrganismsHeader />
    <main>
      <slot />  <!-- содержимое страницы -->
    </main>
    <OrganismsFooter />
  </div>
</template>
