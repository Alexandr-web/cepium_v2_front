# Plugins в Nuxt 4

Папка `app/plugins/` предназначена для выполнения кода при инициализации приложения до монтирования корневого компонента. Плагины позволяют регистрировать глобальные компоненты, добавлять библиотеки, настраивать клиенты API, WebSocket-соединения, инициализировать хранилища и многое другое.

## Основные концепции

- **Автоматическая загрузка** – все файлы в `plugins/` обрабатываются Nuxt автоматически.
- **Контроль порядка** – можно задать порядок выполнения с помощью префиксов (например, `1.api.ts`, `2.pinia.ts`).
- **Клиентские/серверные плагины** – выполнение только на нужной стороне.
- **Доступ к Nuxt-контексту** – плагины получают экземпляр `nuxtApp` и могут расширять его.

## Создание плагина

Файл в `plugins/` должен экспортировать функцию (по умолчанию), которая будет вызвана при старте приложения.

### Пример простого плагина: `plugins/hello.ts`

```ts
export default defineNuxtPlugin(() => {
  return {
    provide: {
      hello: (msg: string) => console.log(`Hello, ${msg}!`);
    }
  };
});
