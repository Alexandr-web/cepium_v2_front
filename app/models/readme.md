# Models - классы для работы с данными от бэкенда

Папка `app/models/` содержит классы, которые описывают бизнес-сущности проекта (пользователя, ордер, позицию, стратегию и т.д.). Эти классы используются для:

- **Инкапсуляции** логики, связанной с конкретной сущностью (валидация, форматирование, вычисляемые свойства).
- **Сериализации / десериализации** данных, получаемых от API бэкенда.
- **Единообразного доступа** к данным во всём проекте.

## Основной принцип

Данные приходят с бэкенда в виде «сырых» объектов (JSON). В проекте эти объекты превращаются в экземпляры соответствующих классов-моделей. Это даёт следующие преимущества:

- **Логика привязана к данным** – например, у ордера есть метод `getTotal()` или `isFilled()`.
- **Простая типизация** – экземпляр класса уже содержит все поля и методы.
- **Переиспользование** – модель можно использовать и в компонентах, и в хранилищах, и в утилитах.
- **Тестируемость** – легко создать фабричные функции для тестовых данных.
