# Nuxt Server Directory

В этой директории находится серверная часть (бэкенд) вашего Nuxt приложения, работающая на базе серверного движка **Nitro**. Все файлы здесь автоматически компилируются в изолированные API-эндпоинты и серверные обработчики.

## Структура папок

* **`/api`** — Маршруты, которые автоматически получают префикс `/api` в URL.
* **`/routes`** — Серверные маршруты без префикса `/api` (например, для создания RSS, sitemap или кастомных эндпоинтов).
* **`/middleware`** — Серверные промежуточные обработчики. Запускаются на *каждый* входящий запрос перед обработкой маршрутов.
* **`/plugins`** — Плагины Nitro для расширения возможностей сервера или подключения к базам данных при старте.
* **`/utils`** — Переиспользуемые серверные утилиты (автоматически импортируются во всей папке `server`).

## Базовый пример эндпоинта

Чтобы создать API-метод `/api/hello`, добавьте файл `server/api/hello.ts`:

```typescript
export default defineEventHandler((event) => {
  return {
    status: 'success',
    message: 'Привет из Nuxt Server!'
  }
})
```

## Основные правила разработки

### 1. Работа с HTTP методами
Используйте суффиксы в названиях файлов для явного указания HTTP-метода:
* `server/api/users.get.ts` — обработает только `GET /api/users`
* `server/api/users.post.ts` — обработает только `POST /api/users`

### 2. Динамические параметры
Для создания динамических маршрутов используйте квадратные скобки:
* `server/api/users/[id].ts` (внутри вызывайте `event.context.params.id`)

### 3. Чтение данных из запроса
Используйте встроенные утилиты h3 / Nitro внутри `defineEventHandler`:
* `getQuery(event)` — для получения query-параметров (`?search=vue`)
* `readBody(event)` — для чтения тела запроса (в `POST`/`PUT` методах)
* `getCookie(event, 'name')` — для чтения куки

### 4. Обработка ошибок
Для возврата HTTP-ошибок используйте `createError`:
```typescript
throw createError({
  statusCode: 401,
  statusMessage: 'Unauthorized',
})
```

## Важное примечание
Код в этой директории выполняется **строго на сервере**. Здесь нельзя использовать браузерные API (такие как `window`, `document` или `localStorage`).
